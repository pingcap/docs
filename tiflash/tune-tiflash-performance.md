# 调优 TiFlash 性能 {#tune-tiflash-performance}

本文介绍了如何通过合理规划机器资源和调优 TiDB 参数来提升 TiFlash 的性能。按照这些方法操作后，你的 TiFlash 集群可以实现最佳性能。

## 规划资源 {#plan-resources}

如果你希望节省机器资源且对隔离没有要求，可以采用同时部署 TiKV 和 TiFlash 的方式。建议你分别为 TiKV 和 TiFlash 预留足够的资源，并且不要共用磁盘。

## 调优 TiDB 参数 {#tune-tidb-parameters}

本节介绍通过调优 TiDB 参数来提升 TiFlash 性能的方法，包括：

-   [强制启用 MPP 模式](#forcibly-enable-the-mpp-mode)
-   [将聚合函数下推到 `Join` 或 `Union` 之前](#push-down-aggregate-functions-to-a-position-before-join-or-union)
-   [启用 `Distinct` 优化](#enable-distinct-optimization)
-   [使用 `ALTER TABLE ... COMPACT` 语句压缩数据](#compact-data-using-the-alter-table--compact-statement)
-   [用 Broadcast Hash Join 替代 Shuffled Hash Join](#replace-shuffled-hash-join-with-broadcast-hash-join)
-   [设置更高的执行并发度](#set-a-greater-execution-concurrency)
-   [配置 `tiflash_fine_grained_shuffle_stream_count`](#configure-tiflash_fine_grained_shuffle_stream_count)

### 强制启用 MPP 模式 {#forcibly-enable-the-mpp-mode}

MPP 执行计划可以充分利用分布式计算资源，从而显著提升批量数据查询的效率。当优化器未为某个查询生成 MPP 执行计划时，你可以强制启用 MPP 模式：

变量 [`tidb_enforce_mpp`](/system-variables.md#tidb_enforce_mpp-new-in-v51) 控制是否忽略优化器的成本估算，强制使用 TiFlash 的 MPP 模式进行查询执行。要强制启用 MPP 模式，运行以下命令：

```sql
set @@tidb_enforce_mpp = ON;
```

以下示例展示了在启用和未启用 `tidb_enforce_mpp` 时的查询结果。未启用前，TiDB 需要从 TiKV 读取数据，并在 TiDB 中执行 `Join` 和 `Aggregation`。启用后，`Join` 和 `Aggregation` 被下推到 TiFlash 执行。此外，由于优化器不一定会生成 MPP 执行计划，通过启用 `tidb_enforce_mpp`，可以强制优化器生成 MPP 执行计划。
