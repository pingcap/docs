# TiDB Cloud Premium の ID アクセス管理 {#identity-access-management-for-tidb-cloud-premium}

このドキュメントでは、 TiDB Cloud Premium の組織および TiDB インスタンス全体でユーザー アクセス、ロール、および権限を管理する方法について説明します。

TiDB Cloud を使用する前に、 [アカウントを登録する](https://tidbcloud.com/free-trial) . メールアドレスとパスワードで[TiDB Cloudでパスワードを管理する](/tidb-cloud/tidb-cloud-password-authentication.md)にサインアップするか、Google、GitHub、または Microsoft アカウントを選択してTiDB Cloudへのシングル サインオン (SSO) を行うことができます。

## 組織とTiDBインスタンス {#organizations-and-tidb-instances}

TiDB Cloud Premiumは、組織とインスタンスの階層構造を採用しており、ユーザーとTiDBインスタンスを効率的に管理できます`Organization Owner`として、組織内に複数のインスタンスを作成し、管理できます。

例えば：

    - Your organization
        - TiDB instance 1
        - TiDB instance 2
        - TiDB instance 3
        ...

この構造では、

-   ユーザーは、組織のメンバーである場合にのみ組織にアクセスできます。
-   TiDB インスタンスにアクセスするには、ユーザーは少なくとも組織内のそのインスタンスに対する読み取り権限を持っている必要があります。

ユーザーの役割と権限の詳細については、 [ユーザーロール](#user-roles)参照してください。

### 組織 {#organizations}

組織には複数の TiDB インスタンスを含めることができます。

TiDB Cloud は組織レベルで課金を計算し、課金の詳細はインスタンスごとに確認できます。

`Organization Owner`の場合、組織内で完全な管理者権限を持ちます。

たとえば、次の操作を実行できます。

-   さまざまな目的に合わせて TiDB インスタンスを作成します。
-   組織レベルとインスタンス レベルのロールを異なるユーザーに割り当てます。
-   タイムゾーンなどの組織全体の設定を構成します。

### TiDBインスタンス {#tidb-instances}

`Instance Manager`の場合は、特定の TiDB インスタンスの設定と操作を管理できます。

たとえば、次の操作を実行できます。

-   不要になった TiDB インスタンスを削除します。
-   必要に応じてインスタンス構成を変更します。

## ユーザーロール {#user-roles}

TiDB Cloud は、組織レベルと TiDB インスタンス レベルの両方で権限を制御するために、さまざまなユーザー ロールを定義します。

組織レベルまたはTiDBインスタンスレベルでユーザーにロールを付与できます。最小限の権限でアクセスできるようにし、セキュリティを維持するために、階層を慎重に計画することをお勧めします。

### 組織の役割 {#organization-roles}

組織レベルでは、 TiDB Cloud は次のロールを定義しており、 `Organization Owner`を招待したり、メンバーに組織ロールを付与したりできます。

| ------------------------------------------------------------------------------- | -------------------- | ------------------------------ | ----------------------------- | ------------------------------------ | --------------------- |
| TiDB インスタンス、API キー、タイムゾーンなどの組織設定を管理します。                                         | ✅                    | ❌                              | ❌                             | ❌                                    | ❌                     |
| 組織のメンバーを追加または削除し、組織の役割を編集します。                                                   | ✅                    | ❌                              | ❌                             | ❌                                    | ❌                     |
| 組織内のすべての TiDB インスタンスに対する権限は`Instance Manager` 。                                 | ✅                    | ❌                              | ❌                             | ❌                                    | ❌                     |
| 組織の支払い情報を管理します。                                                                 | ✅                    | ✅                              | ❌                             | ❌                                    | ❌                     |
| 請求内容をビュー、 [コストエクスプローラー](/tidb-cloud/tidb-cloud-billing.md#cost-explorer)を使用します。 | ✅                    | ✅                              | ✅                             | ❌                                    | ❌                     |

| 組織のすべてのメンバーをビュー。                                                                | ✅                    | ❌                              | ❌                             | ❌                                    | ❌                     |

> **注記：**
>
> -   `Organization Console Audit Manager`ロールは、データベース監査ログではなく、 TiDB Cloudコンソールの監査ログのみを管理します。

### TiDBインスタンスロール {#tidb-instance-roles}

TiDB インスタンス レベルでは、 TiDB Cloud は`Instance Manager`と`Instance Viewer` 2 つのロールを定義します。

> **注記：**
>
> -   `Organization Owner` 、組織内のすべてのインスタンスの`Instance Manager`権限をすべて自動的に継承します。
> -   各 TiDB インスタンス ロールは、デフォルトで`Organization Viewer`ロールのすべての権限を継承します。
> -   組織内のメンバーに TiDB インスタンス ロールがない場合、そのメンバーは組織内のどの TiDB インスタンスにもアクセスできません。

| 許可                                                                                          | `Instance Manager` | `Instance Viewer` |
| ------------------------------------------------------------------------------------------- | ------------------ | ----------------- |
| TiDBインスタンス設定を管理する                                                                           | ✅                  | ❌                 |
| TiDB インスタンスの[データベース監査ログ](/tidb-cloud/tidb-cloud-auditing.md)管理します。                          | ✅                  | ❌                 |
| TiDB インスタンスの作成、変更、削除などの TiDB インスタンス操作を管理します。                                                | ✅                  | ❌                 |
| データのインポート、データのバックアップと復元、データの移行など、TiDB インスタンス データを管理します。                                     | ✅                  | ❌                 |
| [チェンジフィード](/tidb-cloud/changefeed-overview.md)管理します。                                        | ✅                  | ❌                 |
| TiDB インスタンスのルート パスワードを確認してリセットします。                                                          | ✅                  | ❌                 |
| TiDB インスタンスの概要、バックアップ レコード、メトリック、イベント、および[チェンジフィード](/tidb-cloud/changefeed-overview.md)ビュー。 | ✅                  | ✅                 |

## 組織のアクセスを管理する {#manage-organization-access}

### 組織のビューと切り替え {#view-and-switch-between-organizations}

組織を表示したり切り替えたりするには、次の手順を実行します。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左上隅のコンボボックスをクリックします。所属する組織のリストが表示されます。

    > **ヒント：**
    >
    > -   現在特定の TiDB インスタンスのページにいる場合は、左上隅のコンボ ボックスをクリックした後、コンボ ボックス内の ← をクリックして組織リストに戻る必要があります。
    > -   複数の組織のメンバーである場合は、コンボ ボックス内の対象の組織名をクリックして、組織間でアカウントを切り替えることができます。

2.  組織 ID やタイム ゾーンなどの組織の詳細情報を表示するには、組織名をクリックし、左側のナビゲーション ペインで**[組織設定]** &gt; **[全般]**をクリックします。

### 組織のタイムゾーンを設定する {#set-the-time-zone-for-your-organization}

ロール`Organization Owner`の場合、タイム ゾーンに応じてシステムの表示時刻を変更できます。

ローカルタイムゾーン設定を変更するには、次の手順を実行します。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左上隅のコンボ ボックスを使用して対象の組織に切り替えます。

2.  左側のナビゲーション ペインで、 **[組織設定]** &gt; **[全般]**をクリックします。

3.  「**タイムゾーン」**セクションで、ドロップダウンリストからタイムゾーンを選択します。

4.  **[更新]**をクリックします。

### 組織にユーザーを招待する {#invite-a-user-to-your-organization}

`Organization Owner`ロールの場合は、ユーザーを組織に招待できます。

> **注記：**
>
> 必要に応じて直接[TiDBインスタンスにアクセスまたは管理するユーザーを招待する](#invite-a-user-to-access-or-manage-a-tidb-instance)実行することもできます。これにより、ユーザーを組織のメンバーにすることもできます。

ユーザーを組織に招待するには、次の手順を実行します。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左上隅のコンボ ボックスを使用して対象の組織に切り替えます。

2.  左側のナビゲーション ペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **[ユーザー]**ページで、 **[ユーザーを招待]**をクリックします。

4.  招待するユーザーのメール アドレスを入力し、ユーザーの組織ロールを選択します。

    > **ヒント：**
    >
    > -   組織レベルでのデフォルトの役割は`Organization Viewer`です。
    > -   一度に複数のユーザーを招待する場合は、複数のメールアドレスを入力できます。
    > -   招待されたユーザーは、デフォルトではTiDBインスタンスにアクセスできません。ユーザーにTiDBインスタンスへのアクセス権限を付与するには、 [TiDBインスタンスにアクセスまたは管理するユーザーを招待する](#invite-a-user-to-access-or-manage-a-tidb-instance)参照してください。

5.  ユーザーに組織ロールのみを割り当てる必要があり、プロジェクト ロールまたは TiDB インスタンス ロールを割り当てる必要がない場合は、 **[プロジェクトとインスタンスへのアクセスの追加]**オプションを無効にします。

6.  **「招待」**をクリックします。すると、新しいユーザーがユーザーリストに追加されます。同時に、招待されたメールアドレスに確認リンクが記載されたメールが送信されます。

7.  このメールを受信した後、ユーザーはメール内のリンクをクリックして本人確認を行う必要があり、新しいページが表示されます。

8.  招待された電子メール アドレスがTiDB Cloudアカウントのサインアップに使用されていない場合、ユーザーはサインアップ ページに移動してアカウントを作成します。

> **注記：**
>
> メール内の確認リンクは24時間で有効期限が切れます。招待したいユーザーがメールを受信していない場合は、 **「再送信」**をクリックしてください。

### 組織の役割を変更する {#modify-organization-roles}

`Organization Owner`ロールの場合は、組織内のすべてのメンバーの組織ロールを変更できます。

メンバーの組織の役割を変更するには、次の手順を実行します。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左上隅のコンボ ボックスを使用して対象の組織に切り替えます。

2.  左側のナビゲーション ペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **[ユーザー]**ページで、対象メンバーの行の**[...]** &gt; **[ロールの編集]**をクリックします。

### 組織メンバーを削除する {#remove-an-organization-member}

`Organization Owner`ロールの場合は、組織から組織メンバーを削除できます。

組織からメンバーを削除するには、次の手順を実行します。

> **注記：**
>
> 組織からメンバーが削除されると、そのメンバーの TiDB インスタンス アクセスも削除されます。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左上隅のコンボ ボックスを使用して対象の組織に切り替えます。

2.  左側のナビゲーション ペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  [**ユーザー]**ページで、対象メンバーの行の**[...]** &gt; **[削除]**をクリックします。

## TiDBインスタンスアクセスの管理 {#manage-tidb-instance-access}

### TiDBインスタンスにアクセスまたは管理するユーザーを招待する {#invite-a-user-to-access-or-manage-a-tidb-instance}

`Organization Owner`ロールの場合は、TiDB インスタンスにアクセスしたり管理したりするためにユーザーを招待できます。

> **注記：**
>
> 組織外のユーザーを TiDB インスタンスにアクセスまたは管理するために招待すると、そのユーザーは自動的に組織にも参加します。

TiDB インスタンスにアクセスしたり管理したりするためにユーザーを招待するには、次の手順を実行します。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左上隅のコンボ ボックスを使用して対象の組織に切り替えます。

2.  左側のナビゲーション ペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **[ユーザー]**ページで、 **[ユーザーを招待]**をクリックします。

4.  招待するユーザーのメール アドレスを入力し、ユーザーの組織ロールを選択します。

5.  **プロジェクトとインスタンスのアクセスの追加**オプションが有効になっていることを確認し、**インスタンス アクセス**セクションで**アクセスの追加を**クリックして、ユーザーの TiDB インスタンス ロールを選択します。

6.  Click **Add access**. Then the new user is successfully added into the user list. At the same time, an email is sent to the invited email address with a verification link.

7.  このメールを受信した後、ユーザーはメール内のリンクをクリックして本人確認を行う必要があり、新しいページが表示されます。

8.  招待された電子メール アドレスがTiDB Cloudアカウントにサインアップされていない場合、ユーザーはアカウントを作成するためのサインアップ ページに移動します。

> **注記：**
>
> メール内の確認リンクは24時間で有効期限が切れます。ユーザーがメールを受信していない場合は、 **「再送信」**をクリックしてください。

### TiDBインスタンスのロールを変更する {#modify-tidb-instance-roles}

`Organization Owner`ロールの場合は、組織内のすべての組織メンバーの TiDB インスタンス ロールを変更できます。

メンバーの TiDB インスタンス ロールを変更するには、次の手順を実行します。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左上隅のコンボ ボックスを使用して対象の組織に切り替えます。

2.  左側のナビゲーション ペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **[ユーザー]**ページで、対象メンバーの**[...]** &gt; **[ロールの編集]**をクリックします。

## ユーザープロファイルを管理する {#manage-user-profiles}

TiDB Cloudでは、名、姓、電話番号などのプロフィールを簡単に管理できます。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、<mdsvgicon name="icon-top-account-settings">左下隅にあります。</mdsvgicon>

2.  **[アカウント設定]**をクリックします。

3.  表示されたダイアログでプロファイル情報を更新し、 **[更新]**をクリックします。
