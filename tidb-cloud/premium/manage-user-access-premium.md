# TiDB Cloud Premium の ID アクセス管理 {#identity-access-management-for-tidb-cloud-premium}

このドキュメントでは、組織全体およびTiDB Cloud Premiumインスタンスにおけるユーザーアクセス、役割、および権限の管理方法について説明します。

TiDB Cloud を使用する前に、[アカウント登録](https://tidbcloud.com/free-trial)行ってください。電子メールとパスワードを使用して[TiDB Cloudでパスワードを管理する](/tidb-cloud/tidb-cloud-password-authentication.md)でパスワードを管理するか、 TiDB Cloudへのシングル サインオン (SSO) 用に Google、GitHub、または Microsoft アカウントを選択できます。

## 組織と事例 {#organizations-and-instances}

TiDB Cloud Premium は、組織とインスタンスの階層構造を使用して、ユーザーとTiDB Cloud Premium インスタンスを効率的に管理できるようにします。 `Organization Owner`として、組織内で複数のインスタンスを作成および管理できます。

例えば：

    - Your organization
        - TiDB Cloud Premium instance 1
        - TiDB Cloud Premium instance 2
        - TiDB Cloud Premium instance 3
        ...

この構造では：

-   ユーザーは、その組織のメンバーである場合にのみ、その組織にアクセスできます。
-   TiDB Cloud Premiumインスタンスにアクセスするには、ユーザーは組織内でそのインスタンスに対する少なくとも読み取り権限を持っている必要があります。

ユーザーの役割と権限の詳細については、[ユーザーロール](#user-roles)を参照してください。

### 組織 {#organizations}

組織は複数のTiDB Cloud Premiumインスタンスを含めることができます。

TiDB Cloudは組織レベルで課金計算を行い、各インスタンスごとに課金の詳細を確認できます。

`Organization Owner`の場合、組織内で完全な管理者権限が付与されます。

例えば、次のようなことができます。

-   さまざまな目的に合わせて、 TiDB Cloud Premiumインスタンスを作成します。
-   組織レベルおよびインスタンスレベルの役割を、異なるユーザーに割り当てます。
-   タイムゾーンなど、組織全体の各種設定を構成します。

### TiDB Cloud Premiumインスタンス {#tidb-cloud-premium-instances}

`Instance Manager`の場合、特定のTiDB Cloud Premium インスタンスの設定と操作を管理できます。

例えば、次のようなことができます。

-   TiDB Cloud Premiumインスタンスが不要になったら、削除してください。
-   必要に応じてインスタンス構成を変更してください。

## ユーザーロール {#user-roles}

TiDB Cloudは、組織レベルとインスタンスレベルの両方で権限を制御するために、さまざまなユーザーロールを定義しています。

組織レベルまたはインスタンスレベルでユーザーにロールを付与できます。最小権限アクセスを確保し、セキュリティを維持するために、階層構造を慎重に計画することをお勧めします。

### 組織における役割 {#organization-roles}

組織レベルでは、 TiDB Cloud は次の役割を定義しており、 `Organization Owner`はメンバーを招待したり、メンバーに組織の役割を付与したりできます。

| 許可                                                                              | `Organization Owner` | `Organization Billing Manager` | `Organization Billing Viewer` | `Organization Console Audit Manager` | `Organization Viewer` |
| ------------------------------------------------------------------------------- | -------------------- | ------------------------------ | ----------------------------- | ------------------------------------ | --------------------- |
| TiDB Cloud Premiumインスタンス、APIキー、タイムゾーンなどの組織設定を管理します。                             | ✅                    | ❌                              | ❌                             | ❌                                    | ❌                     |
| 組織メンバーの追加または削除、および組織の役割の編集を行います。                                                | ✅                    | ❌                              | ❌                             | ❌                                    | ❌                     |
| 組織内のすべてのTiDB Cloud Premium インスタンスに対する`Instance Manager`権限。                      | ✅                    | ❌                              | ❌                             | ❌                                    | ❌                     |
| 組織の支払い情報を管理する。                                                                  | ✅                    | ✅                              | ❌                             | ❌                                    | ❌                     |
| 請求内容をビュー、 [コストエクスプローラー](/tidb-cloud/tidb-cloud-billing.md#cost-explorer)を使用します。 | ✅                    | ✅                              | ✅                             | ❌                                    | ❌                     |
| 組織の [コンソール監査ログ](/tidb-cloud/tidb-cloud-console-auditing.md)を管理します。              | ✅                    | ❌                              | ❌                             | ✅                                    | ❌                     |
| 組織のメンバー全員をビュー。                                                                  | ✅                    | ❌                              | ❌                             | ❌                                    | ❌                     |
| 組織名とタイムゾーンをビュー。                                                                 | ✅                    | ✅                              | ✅                             | ✅                                    | ✅                     |

> **注記：**
>
> -   `Organization Console Audit Manager`ロールは、データベース監査ログではなく、 TiDB Cloudコンソールでの監査ログのみを管理します。

### インスタンスロール {#instance-roles}

インスタンスレベルでは、 TiDB Cloud は`Instance Manager`と`Instance Viewer`の 2 つの役割を定義します。

> **注記：**
>
> -   `Organization Owner`は、組織内のすべてのインスタンスに対して、 `Instance Manager`すべての権限を自動的に継承します。
> -   各インスタンスロールは、デフォルトで`Organization Viewer`ロールのすべての権限を継承します。
> -   組織内のメンバーにインスタンスロールが付与されていない場合、そのメンバーは組織内のTiDB Cloud Premiumインスタンスにアクセスできません。

| 許可                                                                                                       | `Instance Manager` | `Instance Viewer` |
| -------------------------------------------------------------------------------------------------------- | ------------------ | ----------------- |
| インスタンス設定の管理                                                                                              | ✅                  | ❌                 |
| TiDB Cloud Premium インスタンスのデータベース[データベース監査ログ](/tidb-cloud/tidb-cloud-auditing.md)管理します。                   | ✅                  | ❌                 |
| TiDB Cloud Premiumインスタンスの作成、変更、削除など、インスタンス操作を管理します。                                                      | ✅                  | ❌                 |
| TiDBデータの管理（データインポート、データバックアップと復元、データ移行など）。                                                               | ✅                  | ❌                 |
| [変更フィード](/tidb-cloud/changefeed-overview.md)を管理します。                                                      | ✅                  | ❌                 |
| TiDB Cloud Premiumインスタンスのルートパスワードを確認し、必要に応じてリセットしてください。                                                  | ✅                  | ❌                 |
| TiDB Cloud Premium インスタンスの概要、バックアップ レコード、メトリクス、イベント、および[変更フィード](/tidb-cloud/changefeed-overview.md)をビュー。 | ✅                  | ✅                 |

## 組織のアクセスを管理する {#manage-organization-access}

### 組織をビューおよび切り替える {#view-and-switch-between-organizations}

組織を表示したり、組織を切り替えるには、以下の手順に従ってください。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左上隅のコンボボックスをクリックします。所属する組織の一覧が表示されます。

    > **ヒント：**
    >
    > -   現在特定のTiDB Cloud Premiumインスタンスのページにいる場合、左上隅のコンボボックスをクリックした後、組織リストに戻るには、コンボボックス内の**「マイTiDBに戻る」を**クリックする必要があります。
    > -   複数の組織に所属している場合は、コンボボックス内の対象組織名をクリックすることで、組織間でアカウントを切り替えることができます。

2.  組織の詳細情報（組織IDやタイムゾーンなど）を表示するには、組織名をクリックし、左側のナビゲーションペインで**「組織設定」** ＞ **「一般」**をクリックします。

### 組織のタイムゾーンを設定してください {#set-the-time-zone-for-your-organization}

`Organization Owner`役割を担っている場合は、タイムゾーンに合わせてシステム表示時刻を変更できます。

現地のタイムゾーン設定を変更するには、以下の手順に従ってください。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)では、左上隅のコンボボックスを使用して、対象の組織に切り替えてください。

2.  左側のナビゲーションペインで、 **[組織設定]** &gt; **[一般]**をクリックします。

3.  「**タイムゾーン」**セクションで、ドロップダウンリストからご自身のタイムゾーンを選択してください。

4.  **「更新」**をクリックしてください。

### ユーザーを組織に招待する {#invite-a-user-to-your-organization}

`Organization Owner`の役割をお持ちの場合は、組織にユーザーを招待できます。

> **注記：**
>
> 必要に応じて[TiDB Cloud Premiumインスタンスへのアクセスまたは管理を行うユーザーを招待する](#invite-a-user-to-access-or-manage-a-premium-instance)こともできます。これにより、ユーザーは組織のメンバーになります。

組織にユーザーを招待するには、以下の手順に従ってください。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)では、左上隅のコンボボックスを使用して、対象の組織に切り替えてください。

2.  左側のナビゲーションペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **ユーザー**ページで、 **「ユーザーを招待」**をクリックします。

4.  招待するユーザーのメールアドレスを入力し、そのユーザーの組織内役割を選択してください。

    > **ヒント：**
    >
    > -   組織レベルでのデフォルトの役割は`Organization Viewer`です。
    > -   複数のユーザーを一度に招待したい場合は、複数のメールアドレスを入力できます。
    > -   招待されたユーザーは、デフォルトではTiDB Cloud Premium インスタンスにアクセスできません。ユーザーにインスタンス権限を付与するには、 [TiDB Cloud Premiumインスタンスへのアクセスまたは管理を行うユーザーを招待する](#invite-a-user-to-access-or-manage-a-premium-instance)参照してください。

5.  ユーザーに組織ロールのみを割り当て、プロジェクトロールやインスタンスロールを割り当てる必要がない場合は、 **「プロジェクトとインスタンスへのアクセス権の追加」**オプションを無効にしてください。

6.  **「招待」**をクリックしてください。すると、新しいユーザーがユーザーリストに正常に追加されます。同時に、招待されたメールアドレスに確認リンク付きのメールが送信されます。

7.  このメールを受け取った後、ユーザーはメール内のリンクをクリックして本人確認を行う必要があり、すると新しいページが表示されます。

8.  招待されたメールアドレスがTiDB Cloudアカウントの登録に使用されていない場合、ユーザーはアカウント作成のための登録ページに誘導されます。

> **注記：**
>
> メールに記載されている確認リンクは24時間で有効期限が切れます。招待したいユーザーがメールを受信して​​いない場合は、 **「再送信」**をクリックしてください。

### 組織の役割を変更する {#modify-organization-roles}

`Organization Owner`の役割を持っている場合、組織内のすべてのメンバーの組織役割を変更できます。

メンバーの組織内役割を変更するには、以下の手順に従ってください。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)では、左上隅のコンボボックスを使用して、対象の組織に切り替えてください。

2.  左側のナビゲーションペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **ユーザー**ページで、対象メンバーの行にある**「…」** &gt; **「役割の編集」を**クリックします。

### 組織メンバーを削除する {#remove-an-organization-member}

`Organization Owner`の役割を持っている場合、組織から組織メンバーを削除できます。

組織からメンバーを削除するには、以下の手順に従ってください。

> **注記：**
>
> 組織からメンバーが削除されると、そのメンバーのインスタンスへのアクセス権も削除されます。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)では、左上隅のコンボボックスを使用して、対象の組織に切り替えてください。

2.  左側のナビゲーションペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **ユーザー**ページで、対象メンバーの行にある**[...]** &gt; **[削除]**をクリックします。

## インスタンスへのアクセスを管理する {#manage-instance-access}

### TiDB Cloud Premiumインスタンスへのアクセスまたは管理を行うユーザーを招待する {#invite-a-user-to-access-or-manage-a-premium-instance} {#invite-a-user-to-access-or-manage-a-premium-instance}

`Organization Owner`ロールをお持ちの場合は、ユーザーを招待してTiDB Cloud Premium インスタンスへのアクセスや管理を許可することができます。

> **注記：**
>
> 組織外のユーザーを招待してTiDB Cloud Premiumインスタンスへのアクセスや管理を許可すると、そのユーザーは自動的に組織にも参加することになります。

TiDB Cloud Premiumインスタンスへのアクセスまたは管理をユーザーに許可するには、以下の手順を実行してください。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)では、左上隅のコンボボックスを使用して、対象の組織に切り替えてください。

2.  左側のナビゲーションペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **ユーザー**ページで、 **「ユーザーを招待」**をクリックします。

4.  招待するユーザーのメールアドレスを入力し、そのユーザーの組織内役割を選択してください。

5.  **「プロジェクトとインスタンスへのアクセス権の追加」**オプションが有効になっていることを確認し、 **「インスタンスアクセス」**セクションで**「アクセス権の追加」**をクリックして、ユーザーのインスタンスロールを選択します。

6.  **「アクセス権を追加」**をクリックしてください。すると、新しいユーザーがユーザーリストに正常に追加されます。同時に、招待されたメールアドレスに確認リンク付きのメールが送信されます。

7.  このメールを受け取った後、ユーザーはメール内のリンクをクリックして本人確認を行う必要があり、すると新しいページが表示されます。

8.  招待されたメールアドレスがTiDB Cloudアカウントに登録されていない場合、ユーザーはアカウント作成のための登録ページに誘導されます。

> **注記：**
>
> メールに記載されている確認リンクは24時間後に期限切れとなります。ユーザーがメールを受信しない場合は、 **「再送信」**をクリックしてください。

### インスタンスロールを変更する {#modify-instance-roles}

`Organization Owner`ロールに属している場合、組織内のすべての組織メンバーのインスタンスロールを変更できます。

メンバーのインスタンスロールを変更するには、以下の手順を実行します。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)では、左上隅のコンボボックスを使用して、対象の組織に切り替えてください。

2.  左側のナビゲーションペインで、 **[組織設定]** &gt; **[ユーザー]**をクリックします。

3.  **ユーザー**ページで、対象メンバーの**「...」** &gt; **「役割の編集」**をクリックします。

## ユーザープロファイルの管理 {#manage-user-profiles}

TiDB Cloudでは、氏名（姓、名）、電話番号など、プロフィール情報を簡単に管理できます。

1.  [TiDB Cloudコンソール](https://tidbcloud.com)で、左下隅にある<MDSvgIcon name="icon-top-account-settings" />をクリックします。

2.  **アカウント設定**をクリックしてください。

3.  表示されたダイアログで、プロファイル情報を更新し、 **「更新」**をクリックします。
